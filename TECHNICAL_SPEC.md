# 📱 สถาปัตยกรรมระบบและเอกสารข้อกำหนดทางเทคนิค (Technical Specification & Architecture)
## ระบบสารสนเทศ อวท. วิทยาลัยเทคนิคอุดรธานี

เอกสารฉบับนี้จัดทำขึ้นเพื่อระบุโครงสร้างซอร์สโค้ด โมดูล ส่วนประสานงานผู้ใช้ (UI/UX) และการกำหนดค่าหน้าเริ่มต้น (Default Page) ของระบบ อวท. วิทยาลัยเทคนิคอุดรธานี ตามข้อกำหนดและลักษณะของส่วนประกอบที่ออกแบบไว้

## 🎨 1. ข้อกำหนดการออกแบบปุ่มตามภาพตัวอย่าง (UI Button Specifications)
อ้างอิงจากภาพเมนูของแผนกวิชาต่างๆ ตัวปุ่มถูกออกแบบให้มีความทันสมัย เรียบง่าย และมีมิติที่ชัดเจน โดยมีรายละเอียดสีและสไตล์ที่นำไปใช้พัฒนาต่อด้วย CSS / Tailwind CSS ดังนี้:

### 📊 โทนสีที่ใช้งาน (Color Palette)
*   **สีพื้นหลังของปุ่ม (Background Color):** `#2B88F7` (Dodger Blue / Bright Cerulean) เป็นโทนสีฟ้าสดใสที่ให้ความรู้สึกน่าเชื่อถือและเป็นเทคโนโลยี
*   **สีของไอคอนและข้อความหลัก (Text & Icon Color):** `#FFFFFF` (Pure White) เพื่อการอ่านที่ชัดเจนและตัดกับพื้นหลังได้ดี
*   **สีของข้อความรองย่อย (Subtext Color):** `#E0EFFF` (Light Ice Blue) หรือสีขาวที่มีค่าความโปร่งใส (Opacity) ประมาณ 80-85% สำหรับส่วน "ดูรายละเอียดแผนกวิชา"
*   **สีพื้นหลังของเว็บรวม (Page Background):** โทนสีขาวสว่างหรือเทาอ่อนมาก ๆ (`#F8FAFC`) ร่วมกับลวดลายโครงข่ายโยงใยสีเทาจาง ๆ (Network node pattern background)
*   **สีเงาของปุ่ม (Box Shadow):** `rgba(43, 136, 247, 0.2)` เป็นเงาสีฟ้าฟุ้งกระจายจางๆ ด้านล่างและด้านข้างเพื่อให้ปุ่มดูลอยมีมิติ

### 📐 โครงสร้างและสัดส่วน (Layout & Dimensions)
*   **รูปทรง (Border Radius):** มีความโค้งมนสูง (`border-radius: 18px` หรือ `rounded-2xl` ใน Tailwind)
*   **การจัดวางภายใน (Flex Layout):** จัดเรียงแบบแถว (Row)
    *   **ฝั่งซ้าย:** ไอคอนรูปบุคคลและเฟืองคู่กัน (`User & Cog Icon`) สีขาวเด่นชัด
    *   **ฝั่งขวา:** จัดเรียงแบบคอลัมน์แนวตั้ง (Flex Column)
        *   **บรรทัดบน:** ชื่อแผนกวิชา (ตัวหนา ขนาดประมาณ `14-16px`)
        *   **บรรทัดล่าง:** ข้อความย่อย "ดูรายละเอียดแผนกวิชา" (ตัวธรรมดา ขนาดเล็กกว่าประมาณ `11-12px`)
*   **เอฟเฟกต์เมื่อนำเมาส์ไปชี้ (Hover Effect):** 
    *   เพิ่มความสว่างของสีพื้นหลังเล็กน้อย (`#4195FF`)
    *   ขยายขนาดปุ่มขึ้นเล็กน้อยด้วยเอฟเฟกต์ดันตัว (`transform: scale(1.02); transition: all 0.2s ease-in-out;`)

## 🏠 2. โครงสร้างหน้าแรกเริ่มต้น (Default Page Layout: HomeView.ts)
ระบบได้รับการตั้งค่าให้ **`HomeView.ts` เป็นหน้าเริ่มต้น (Default Page)** เมื่อผู้ใช้งานเปิดระบบเข้ามา โดยในหน้านี้ประกอบไปด้วยส่วนประกอบสำคัญ 2 ส่วนหลัก:

1.  **📱 ข่าวสารล่าสุด (Latest News Section):** 
    *   พื้นที่แสดงการอัปเดต กิจกรรมด่วน และประกาศสำคัญจากวิทยาลัย
2.  **🌐 Facebook Feed — อวท. วิทยาลัยเทคนิคอุดรธานี:**
    *   ส่วนการเชื่อมต่อและดึงข้อมูลฟีด (Embed Widget) โดยตรงจากหน้าแฟนเพจหลักของ อวท. วิทยาลัยเทคนิคอุดรธานี เพื่อให้ผู้ใช้เข้าถึงข้อมูลบนโซเชียลมีเดียได้ทันทีโดยไม่ต้องเปลี่ยนหน้าต่าง
3.  **🎛️ แผงเมนูปุ่มทางลัด (Shortcut Department Menu):**
    *   รวมปุ่มทางลัดเข้าสู่ระบบข้อมูลของแต่ละแผนกวิชาจำนวน 12 แผนกหลักตามภาพตัวอย่าง (เช่น แผนกวิชาช่างยนต์, แผนกวิชาช่างกลโรงงาน, แผนกวิชาช่างเชื่อมโลหะ, แผนกวิชาช่างไฟฟ้า, แผนกวิชาช่างอิเล็กทรอนิกส์, แผนกวิชาช่างก่อสร้าง-โยธา, แผนกเทคนิคสถาปัตยกรรม, แผนกเทคโนโลยีอุตสาหกรรม, แผนกวิชายนตรกรรมไฟฟ้า, แผนกวิชาการจัดการโลจิสติกส์, แผนกเทคโนโลยีสารสนเทศ และแผนกเทคโนโลยีคอมพิวเตอร์)

## 📂 3. รายละเอียดโครงสร้างไฟล์ส่วนประกอบระบบทั้งหมด (13 Core Files)

นี่คือไฟล์หลักทั้ง 13 ไฟล์ที่ประกอบกันเป็นระบบสารสนเทศ อวท. พร้อมคำอธิบายหน้าที่และฟังก์ชันเฉพาะของแต่ละส่วน:

### 1. 🗂️ `SidebarMenu.ts`
*   **หน้าที่หลัก:** โครงสร้างและตรรกะควบคุมเมนูด้านข้าง (Sidebar Navigation)
*   **คุณลักษณะ:** ออกแบบมาให้ล็อกเกาะติดหน้าจอ (Fixed/Sticky Sidebar) รองรับการยืด-หดตัว (Responsive Collapsible) เพื่อเพิ่มพื้นที่ในการอ่านเอกสารบนหน้าจอขนาดเล็ก

### 2. ⚙️ `SettingsPopup.ts`
*   **หน้าที่หลัก:** กล่องป๊อปอัปสำหรับควบคุมการตั้งค่าและการเข้าถึงของผู้ใช้งาน
*   **ฟังก์ชันภายใน:**
    *   **Dark/Light Mode:** สลับโทนสีของระบบ (สลับสีข้อความ ดำ-ขาว อัตโนมัติ)
    *   **Size/Font Adjustment:** เพิ่ม-ลดขนาดอักษรและเปลี่ยนรูปแบบฟอนต์ตามความเหมาะสมกับสายตา
    *   **Web Push Notification:** เปิด-ปิดการรับแจ้งเตือนแบบพุชผ่านเบราว์เซอร์สำหรับข่าวสารใหม่

### 3. 🏠 `HomeView.ts` *(หน้าเริ่มต้น - Default Page)*
*   **หน้าที่หลัก:** หน้าแรกของระบบสารสนเทศ
*   **เนื้อหาภายใน:** แสดงฟีดข่าวสารล่าสุด, ส่วนเชื่อมต่อ **Facebook Feed — อวท. วิทยาลัยเทคนิคอุดรธานี** และแผงปุ่มกดทางลัดเข้าสู่แผนกวิชาต่างๆ ที่มีลักษณะโค้งมนสีฟ้าสดใสตาม Spec รูปภาพ

### 4. 📊 `DashboardView.ts`
*   **หน้าที่หลัก:** หน้าแสดงผลข้อมูลสถิติภาพรวม (Data Visualization)
*   **เนื้อหาภายใน:** กราฟสรุปจำนวนกิจกรรม, อัตราการส่งเล่มโครงการของแต่ละชมรมวิชาชีพ และตัวเลขสถิติสำคัญที่ช่วยให้ผู้บริหารและอาจารย์มองเห็นภาพรวมของ อวท. ได้อย่างรวดเร็ว

### 5. 📋 `ActivityListView.ts`
*   **หน้าที่หลัก:** ส่วนแสดงรายการกิจกรรมทั้งหมดประจำภาคเรียน
*   **ฟังก์ชันภายใน:** ตารางแสดงชื่อกิจกรรม วันเวลา สถานที่ และสถานะการดำเนินงาน (ยังไม่เริ่ม/กำลังดำเนินการ/เสร็จสิ้น) พร้อมตัวกรองแยกประเภทกิจกรรมตามแผนงาน

### 6. 📑 `ProjectSummaryView.ts`
*   **หน้าที่หลัก:** หน้าสรุปผลการส่งโครงการของนักศึกษา
*   **การอัปเดตโครงสร้างตารางข้อมูล:** มีการเพิ่มคอลัมน์สำคัญเพิ่มเติมนอกเหนือจากคอลัมน์พื้นฐานเพื่อความละเอียดของข้อมูล ดังนี้:
    1.  คอลัมน์ **"ชื่อโครงการ"** (เพื่อระบุรายละเอียดหัวข้อชัดเจน)
    2.  คอลัมน์ **"ชมรมวิชาชีพ/อวท."** (เพื่อจำแนกสังกัดผู้รับผิดชอบอย่างเป็นระบบ)

### 7. 📝 `BookSummaryView.ts`
*   **หน้าที่หลัก:** หน้าตรวจสอบและจัดการสรุปรูปเล่มรายงานโครงการ
*   **ระบบค้นหาขั้นสูง (Advanced Search Panel):** มาพร้อมกับแถบค้นหาแบบพิมพ์ (Text Search Input) ที่เชื่อมต่อทำงานร่วมกัน (Interconnected) กับกล่องเลือกฟิลเตอร์คัดกรอง 3 มิติหลัก:
    *   ฟิลเตอร์แยกตาม **ชมรมวิชาชีพ/อวท.**
    *   ฟิลเตอร์แยกตาม **ภาคเรียน**
    *   ฟิลเตอร์แยกตาม **ปีการศึกษา**

### 8. 📅 `AnnualPlanView.ts`
*   **หน้าที่หลัก:** ปฏิทินและแผนการปฏิบัติกิจกรรมประจำปีขององค์กร
*   **เนื้อหาภายใน:** แผนงาน Timeline (Gantt Chart หรือ List View) แสดงตารางงานตลอดปีการศึกษาของ อวท. เพื่อให้นักศึกษาและอาจารย์วางแผนร่วมกิจกรรมได้ถูกต้อง

### 9. 🏫 `ClubListView.ts`
*   **หน้าที่หลัก:** ฐานข้อมูลรายชื่อชมรมวิชาชีพทั้งหมดในวิทยาลัยเทคนิคอุดรธานี
*   **เนื้อหาภายใน:** ข้อมูลติดต่อ รายชื่อคณะกรรมการชมรมวิชาชีพของแต่ละแผนกวิชา และช่องทางส่งเอกสารเฉพาะชมรม

### 10. 📖 `RegulationView.ts`
*   **หน้าที่หลัก:** หน้าสืบค้นระเบียบและข้อบังคับต่างๆ ของ อวท.
*   **คุณลักษณะและระบบสิทธิ์:**
    *   มีแถบค้นหาข้อมูลคู่กับคอลัมน์แสดง **ปีการศึกษา/ภาคเรียน** เพื่อให้ค้นหาเอกสารเก่า-ใหม่ได้แม่นยำ
    *   **ระบบสิทธิ์การอัปโหลด (Role-based Upload System):** ระบบแยกสิทธิ์สำหรับ นักศึกษา อวท. และอาจารย์ ในการอัปโหลดไฟล์ PDF หรือ Word
    *   **การแสดงผล:** เมื่อคลิกเปิดลิงก์เอกสาร ตัวระบบจะทำการ **เปิดลิงก์ในหน้าต่างใหม่ (target="_blank")** เสมอ เพื่อไม่ให้รบกวนหน้าหลักที่ทำงานอยู่

### 11. 📁 `AftFormsView.ts`
*   **หน้าที่หลัก:** ศูนย์รวมการดาวน์โหลดแบบฟอร์มต่างๆ ของ อวท.
*   **ระบบ Alert แจ้งเตือนความสะดวก (User Experience Notice):** 
    *   มีระบบป๊อปอัประบบเตือนภัยคำแนะนำ: *"กรุณาปิด Sidebar ก่อนดาวน์โหลดแบบฟอร์ม เพื่อความสะดวกในการแสดงผลเอกสาร ควรปิดเมนูด้านข้างก่อนเปิดไฟล์"* โดยตัวหนังสือจะแปรผันตามโหมด (Light Mode=ตัวอักษรดำ / Dark Mode=ตัวอักษรขาว)
*   **บันทึกการซ่อมบำรุงระบบ (Bug Fixing Registry):** 
    *   แก้ไขบั๊กและปรับโครงสร้างการจัดวางสัดส่วนข้อความรหัสสั้น (Short-code) ให้มีความกระชับ เรียบร้อย ไม่ซ้อนทับกันในส่วนของ**แถวที่ 4** และ **แถวที่ 9** ของหน้าฟอร์มเรียบร้อยแล้ว

### 12. 🗂️ `FilesManagementView.ts`
*   **หน้าที่หลัก:** คลังจัดเก็บและบริหารไฟล์ส่วนกลางแบบบูรณาการ
*   **ระบบคัดกรอง (Multi-filtering Dropdown):** ป้องกันความสับสนของไฟล์ปริมาณมากด้วยกล่องรายการตัวเลือกแยกคัดกรองแบบระบุเจาะจง 3 ส่วน:
    1.  แยกตาม **ปีการศึกษา**
    2.  แยกตาม **ภาคเรียน**
    3.  แยกตาม **ประเภทไฟล์** (เช่น .pdf, .docx, .xlsx)

### 13. 🗺️ `CollegeMapView.ts`
*   **หน้าที่หลัก:** แผนที่จำลองสถานที่ภายในวิทยาลัยเทคนิคอุดรธานี
*   **เนื้อหาภายใน:** แผนที่แบบโต้ตอบได้ (Interactive Map) แสดงที่ตั้งของอาคารเรียน ตึกอำนวยการ และโดยเฉพาะ **ห้องสำนักงาน อวท.** เพื่อให้นักศึกษาใหม่หรือบุคคลภายนอกเดินทางมาติดต่อส่งเล่มโครงการได้อย่างถูกต้องและสะดวกสบาย

## 💻 4. โค้ดตัวอย่างการกำหนด Route หน้าแรกเริ่มต้น (Vue Router / React Router Guide)
เพื่อให้หน้าแรก (`HomeView.ts`) ทำหน้าที่เป็นหน้า Default เสมอเมื่อเปิดระบบ สามารถเขียนโครงสร้าง Route ได้ดังนี้:

```typescript
// router/index.ts (ตัวอย่างการคอนฟิกสำหรับระบบ)
import { createRouter, createWebHistory } from 'vue-router';
import HomeView from '../views/HomeView.ts';

const routes = [
  {
    path: '/',
    name: 'home',
    component: HomeView, // ถูกโหลดเป็นหน้าแรกของระบบทันทีเมื่อเข้า URL หลัก
    meta: { defaultPage: true }
  },
  {
    path: '/dashboard',
    name: 'dashboard',
    component: () => import('../views/DashboardView.ts')
  },
  {
    path: '/aft-forms',
    name: 'aft-forms',
    component: () => import('../views/AftFormsView.ts')
  }
  // ... รายการ Route ของไฟล์อื่นๆ ที่เหลือ
];

const router = createRouter({
  history: createWebHistory(),
  routes
});

export default router;
```
